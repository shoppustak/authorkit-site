  <files>index.html, features.html, pricing.html, docs.html</files>

    - /Users/maulik/authorkit-site/includes/header.html (check if fonts are loaded here)
  </read_first>
  <action>
    Update all HTML files with optimized Google Fonts loading:

    For each HTML file (index.html, features.html, pricing.html, docs.html), update the `<head>` section:

    1. Add preconnect hints BEFORE any CSS or font loads (place near top of `<head>`):
    ```html
    <!-- DNS prefetch and preconnect for faster font loading -->
    <link rel="dns-prefetch" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    ```

    2. Update Google Fonts links to include font-display=swap parameter:
    - Find existing Google Fonts `<link>` tags
    - Add `&display=swap` to the URL parameters
    - Example: Change:
      `https://fonts.googleapis.com/css2?family=Playfair+Display:wght@600;700`
      To:
      `https://fonts.googleapis.com/css2?family=Playfair+Display:wght@600;700&display=swap`

    3. For Source Serif 4 font, ensure it also has display=swap:
    ```html
    <link href="https://fonts.googleapis.com/css2?family=Source+Serif+4:opsz,wght@8..60,400;8..60,600&display=swap" rel="stylesheet">
    ```

    4. Add a fallback font style immediately after fonts (prevents layout shift):
    ```html
    <style>
      /* Fallback fonts to prevent layout shift */
      body { font-family: 'Source Serif 4', Georgia, serif; }
      h1, h2, h3, h4, h5, h6, .font-display { font-family: 'Playfair Display', Georgia, serif; }
    </style>
    ```

    This eliminates Flash of Invisible Text (FOIT) and reduces render-blocking time.
  </action>
  <verify>
    <automated>grep -l "display=swap" index.html features.html pricing.html docs.html | wc -l | grep "4"</automated>
  </verify>

    - docs.html contains '&display=swap' in Google Fonts URLs

    For each HTML file, make these changes in the `<head>` section:

    1. Replace multiple CSS file references with single optimized CSS:
    ```html

    <!-- <link rel="stylesheet" href="css/style.css"> -->
    <!-- <link rel="stylesheet" href="css/monochrome-components.css"> -->
    <!-- <link rel="stylesheet" href="css/design-tokens.css"> -->

    <!-- Keep only the main Tailwind CSS (or replace with minified version): -->
    <link rel="stylesheet" href="css/styles.css">
    ```

    2. Add defer attribute to ALL JavaScript files in `<head>`:
    ```html
    <!-- Update script tags to use defer -->
    <script src="js/main.min.js" defer></script>
    <script src="js/header-loader.js" defer></script>
    <script src="js/footer-loader.js" defer></script>
    ```

    3. For any bookshelf-specific pages, also defer those scripts:
    ```html
    <script src="js/bookshelf.min.js" defer></script>
    <script src="js/bookshelf-browse.min.js" defer></script>
    ```

    4. For inline scripts that aren't critical, wrap them to execute after DOM:
    ```html
    <script defer>
      document.addEventListener('DOMContentLoaded', function() {
        // Original inline script content here
      });
    </script>
    ```

    5. Special handling for docs.html (the problematic redirect page):
    - Remove ALL JavaScript except meta refresh
    - The page should only have:
      - Meta refresh tag for redirect
      - Basic CSS for styling the redirect message
      - No JavaScript files at all (they cause the 1224ms blocking time)

    The defer attribute allows scripts to download in parallel but execute in order after HTML parsing completes, eliminating render blocking.
